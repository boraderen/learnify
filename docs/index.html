Learnify API Reference

Readable machine learning from scratch.

Learnify is a didactic Python library for machine learning and deep learning built with raw Python and NumPy. This page documents the current public API that is exported from learnify and meant to be easy to publish with GitHub Pages.

Version 0.1.0 Python >= 3.11 NumPy-backed models Matplotlib plotting helpers

Main export: from learnify import ...

Core areas: autodiff, visualization, optimization, metrics, ML, NN

GitHub Pages: publish the repository's docs/ folder.

Overview

The public API is intentionally small and study-friendly. Models expose familiar fit, predict, and sometimes predict_proba methods, while the autodiff stack centers on a single scalar Value type.

Top-level imports
Value, GradientDescentOptimizer, gradient_descent_step, computation_graph_svg, computation_graph_dot, save_computation_graph_svg, trace_graph, mean_squared_error, accuracy_score, r2_score, rng, plotting, LinearRegressionGD, LogisticRegressionGD, DecisionTreeClassifier, RandomForestClassifier, LinearSVM, KMeans, AgglomerativeClustering, and MLP.

Install And Quick Start

Learnify targets modern Python and keeps setup minimal for local experiments, notebooks, and teaching examples.

Install 
uv sync --dev

Runtime dependencies include numpy and matplotlib.

Computation Graph 
from learnify import Value, computation_graph_svg

x = Value(2.0, label="x")
y = Value(-3.0, label="y")
z = ((x * y) + 4.0).tanh()
z.label = "z"

z.backward()
svg = computation_graph_svg(z)

Autodiff And Graph Utilities

Scalar reverse-mode autodiff powers the neural-network components and the graph visualization helpers.

Class

Value

A scalar node in a computation graph.

Value(data, _children=(), op="", label="")
  • exp(), tanh(), relu(), and sigmoid() create derived nodes.
  • topo() returns a topologically ordered list of connected nodes.
  • zero_grad() clears gradients across the reachable graph.
  • backward(grad=1.0) runs reverse-mode backpropagation from the current node.
Functions

Graph Rendering

trace_graph(root) -> (nodes, edges)
computation_graph_svg(root, *, node_width=200, node_height=94, horizontal_gap=64, vertical_gap=28, padding=24) -> str
save_computation_graph_svg(root, path, **kwargs) -> Path
computation_graph_dot(root) -> str
  • trace_graph walks the graph from leaves to a root value.
  • computation_graph_svg returns a self-contained SVG string.
  • save_computation_graph_svg writes that SVG to disk and returns the output path.
  • computation_graph_dot exports a Graphviz DOT view of the same graph.
Optimization

Gradient Descent

gradient_descent_step(parameters, gradients, learning_rate) -> None
GradientDescentOptimizer(learning_rate=0.01)
  • step_values(parameters) updates a sequence of Value parameters in place.
  • zero_grad(parameters) clears stored gradients on those values.
  • step_arrays(parameters, gradients) applies the same logic to NumPy arrays.

Models

Learnify includes gradient-descent regressors, classical classifiers, clustering algorithms, and a small multilayer perceptron.

Regression

LinearRegressionGD

Linear regression optimized with full-batch gradient descent.

LinearRegressionGD(learning_rate=0.01, n_iterations=1000, fit_intercept=True)
fit(X, y) -> LinearRegressionGD
predict(X) -> np.ndarray
score(X, y) -> float

Classification

Binary logistic regression trained with gradient descent.

Usage Notes

These details are useful when integrating Learnify into examples or teaching material.

Input Conventions

  • Most model APIs accept NumPy arrays and reshape one-dimensional feature arrays into column vectors.
  • LogisticRegressionGD and LinearSVM currently support binary classification only.
  • Several plotting helpers assume one or two input features, depending on the chart type.

Educational Focus

  • Implementations favor readability over scikit-learn compatibility or production optimizations.
  • Many fitted models expose intermediate artifacts such as loss histories, feature importances, and merge traces.
  • The autodiff engine is scalar-first, which keeps the internals easy to inspect and visualize.
